# Code Review Checklist

Use this checklist when reviewing Rust code generated by or modified with rust-forge-skill.

---

## Anti-Patterns (Must Fix â€” zero tolerance)

- [ ] No `.unwrap()` or `.expect()` in library or server code
- [ ] No `unsafe` blocks without `SAFETY` comments
- [ ] No `println!` / `eprintln!` in production code
- [ ] No `static mut` declarations
- [ ] No `anyhow::Error` in library crates (use `thiserror`)
- [ ] No `as` casts without explicit bounds checking
- [ ] No `unwrap()` in test utility functions

---

## Architecture

- [ ] Layer order is correct: `api â†’ service â†’ repository â†’ domain`
- [ ] Dependencies flow inward only (inner layers don't import outer layers)
- [ ] Domain layer has no external I/O dependencies
- [ ] `lib.rs` exposes a clean public API
- [ ] Prelude module exists if it reduces import noise

---

## Ownership & Lifetimes

- [ ] All `Clone` calls are intentional and justified
- [ ] No accidental large clones in hot paths
- [ ] `Arc<State>` used for shared state in async contexts (not `static`)
- [ ] `Rc` not used in multithreaded code without justification
- [ ] Lifetimes are explicit where they can't be elided
- [ ] No dangling references or use-after-free patterns

---

## Error Handling

- [ ] Library code uses `thiserror` for domain errors
- [ ] Application/CLI code uses `anyhow` for boundary errors
- [ ] Error types implement `Debug` and `Clone`
- [ ] Error messages are user-friendly (no internal jargon)
- [ ] Error propagation uses `?` operator throughout
- [ ] No `unwrap()` on `Result`/`Option` in error paths
- [ ] Errors include context (`context()`, `with_context()`)

---

## Async Code

- [ ] Async traits use `async_trait`
- [ ] No blocking operations in async code (use `tokio::task::spawn_blocking`)
- [ ] Connection pools use `PgPoolOptions` with timeouts
- [ ] Graceful shutdown implemented for servers
- [ ] State passed via `Arc<State>`, not global `static`
- [ ] Join handles awaited or detached intentionally
- [ ] No async blocks that immediately `.await` (simplify to sync)

---

## Traits & Generics

- [ ] Trait bounds are explicit and minimal
- [ ] Blanket impls are documented
- [ ] Object safety considered for all traits
- [ ] `Send + Sync` bounds stated where needed
- [ ] Generics don't exceed complexity warranted by use case

---

## Dependency Hygiene

- [ ] No dependency on `unstable` features (except `nightly` crate)
- [ ] Dev dependencies clearly separated from production dependencies
- [ ] No unnecessary transitive dependencies
- [ ] `cargo tree -e features` shows expected feature gates
- [ ] `cargo audit` passes (no known vulnerabilities)

---

## Tests

- [ ] Unit tests for all public functions
- [ ] Integration tests for API/repository layer
- [ ] Error path tests (not just happy path)
- [ ] Tests do not use `.unwrap()` in test utility functions
- [ ] Property-based tests added for parsers/serializers
- [ ] `cargo test --workspace --all-features` passes

---

## Documentation

- [ ] All public functions have doc comments (`///`)
- [ ] `SAFETY` comments on all `unsafe` blocks (explain invariant, why holds, UB consequences)
- [ ] README documents build commands and environment variables
- [ ] `.env.example` lists all required environment variables
- [ ] Complex algorithms have inline comments explaining the why
- [ ] `CHANGELOG.md` updated for user-facing changes

---

## Performance

- [ ] No allocations in hot paths (pre-allocate buffers)
- [ ] Iterators used over `for` loops where applicable
- [ ] Clone avoided in hot paths (borrow instead)
- [ ] No needless `.to_string()` / `format!()` in hot paths
- [ ] String concatenation uses `write!` or `format!` appropriately
- [ ] `cargo bench` results are stable across runs

---

## Unsafe Code

- [ ] Every `unsafe` block has a `SAFETY` comment
- [ ] SAFETY comment states: invariant, why it holds, UB if violated
- [ ] Raw pointer dereferences validated before use
- [ ] No `unsafe` blocks that could be replaced with safe alternatives
- [ ] `static mut` replaced with atomics or `Mutex`
- [ ] `unsafe impl` for `Send`/`Sync` documented with justification

---

## Quality Gate Commands

```bash
cargo fmt --all -- --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test --workspace --all-features
cargo doc --workspace --all-features --no-deps      # libraries only
./scripts/audit_unsafe.sh                          # FFI/WASM only
./scripts/check_msrv.sh                           # libraries only
```

| Gate | Command | Pass Criteria |
|---|---|---|
| Format | `cargo fmt --all -- --check` | No diff |
| Lint | `cargo clippy -- -D warnings` | Zero warnings |
| Tests | `cargo test --workspace --all-features` | All pass |
| Docs | `cargo doc --no-deps` | No broken links |
| Unsafe | `./scripts/audit_unsafe.sh` | All unsafe documented |
| MSRV | `./scripts/check_msrv.sh` | Builds on MSRV toolchain |
