# Agent Notes — FFI Wrapper Template

## Template Purpose
Scaffolds a Rust FFI wrapper crate with C interop via `bindgen`. Use for wrapping C/C++ libraries to call from Rust, or exposing a Rust library to C.

## Instantiation Notes

### Required Customisations
1. **C header** — Replace `src-c/lib.h` with the actual C header from the library you are wrapping
2. **C source** — Replace `src-c/lib.c` with the actual C implementation (or provide your own compiled `.a`/`.so`)
3. **`build.rs`** — Update `staticlib_name` to match your actual library name
4. **`Cargo.toml`** — Set the correct `links` value matching your library's build script

### Module Structure
```
src/
├── lib.rs         — Public Rust API (pub extern "C" fn)
├── wrapper.rs      — FFI-safe wrapper (unsafe boundary)
├── error.rs       — FFI error types (thiserror)
└── bindings.rs    — Auto-generated by bindgen (DO NOT EDIT)
src-c/
├── lib.h          — Your C header
└── lib.c          — Your C source
```

### Critical Rules
- **DO NOT edit `bindings.rs`** — it is auto-generated by bindgen
- After changing `src-c/` files, run `cargo build` to regenerate bindings
- SAFETY comments on every `unsafe` block are **mandatory**
- Use `cbindgen` to generate the inverse Rust→C header if needed

### Validation
```bash
cargo build
./scripts/audit_unsafe.sh    # All unsafe blocks must have SAFETY comments
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```
