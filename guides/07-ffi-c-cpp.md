# 07 — FFI: C and C++ Interoperability

**Purpose:** Guide agents to write safe Rust FFI bindings to C/C++ libraries. FFI is inherently unsafe — every boundary crossing requires rigorous safety guarantees. This guide covers bindgen, cbindgen, safe wrapper patterns, and SAFETY documentation.

---

## When to Use FFI

**Use FFI when:**
- Integrating with existing C/C++ libraries
- Writing performance-critical code in C and wrapping in Rust
- Creating libraries consumed by C/C++ programs
- Accessing system APIs not available in Rust

**Do not use FFI if:**
- A pure-Rust alternative exists (always prefer Rust)
- The complexity of unsafe boundary management is not justified
- The "wrapper" would be thin enough that a header-only C++ template suffices

## build.rs — bindgen + cc

```rust
// build.rs
fn main() {
    // Re-run if header changes
    println!("cargo:rerun-if-changed=src-c/lib.h");

    // Compile the C library
    cc::Build::new()
        .file("src-c/lib.c")
        .compile("my_clib");

    println!("cargo:rustc-link-lib=static=my_clib");

    // Generate Rust bindings from C header
    let bindings = bindgen::Builder::default()
        .header("src-c/lib.h")
        .use_core()
        .prepend_type_name(true)
        .ctypes_prefix("::ctypes")
        .generate()
        .expect("Unable to generate bindings");

    bindings
        .write_to_file("src/bindings.rs")
        .expect("Couldn't write bindings to src/bindings.rs");
}
```

**Cargo.toml for build:**
```toml
[build-dependencies]
bindgen = "0.69"
cc = "1.0"
```

## C Header Design

```c
// src-c/lib.h
#ifndef LIB_H
#define LIB_H

#include <stdint.h>
#include <stddef.h>

// Opaque handle type — caller can't see internals
typedef struct Vec3Handle Vec3Handle;

// Enum for error codes
typedef enum {
    LIB_OK = 0,
    LIB_ERROR_INVALID_INPUT = 1,
    LIB_ERROR_ALLOCATION = 2,
    LIB_ERROR_NULL_PTR = 3,
} LibError;

// Memory management functions
void* lib_alloc(size_t size);
void  lib_free(void* ptr);

// API functions
// All functions that return pointers use output parameters
LibError lib_vec3_create(double x, double y, double z, Vec3Handle** out);
void     lib_vec3_destroy(Vec3Handle* ptr);
double   lib_vec3_length(const Vec3Handle* vec);

#endif // LIB_H
```

**FFI design rules:**
- Use opaque handle types (`typedef struct Handle Handle`) to hide internals
- Return errors via enum, not via NULL returns (unless documented)
- All output parameters are double-indirect (`Type** out`) to allow NULL checking
- Explicit memory allocation/deallocation functions (never mix malloc/free)

## Safe Wrapper Pattern

```rust
// src/wrapper.rs
use std::ptr;
use crate::bindings::{Vec3Handle, LibError};
use crate::error::FfiError;

/// Owns a C Vec3 and manages its lifetime.
/// Drop calls lib_vec3_destroy to free memory.
pub struct Vec3 {
    raw: *mut Vec3Handle,
}

impl Vec3 {
    /// Creates a new Vec3 from x, y, z coordinates.
    ///
    /// # Errors
    ///
    /// Returns `FfiError::AllocationFailed` if C allocation fails.
    /// Returns `FfiError::InvalidInput` if coordinates are invalid.
    pub fn new(x: f64, y: f64, z: f64) -> Result<Self, FfiError> {
        let mut handle_ptr: *mut Vec3Handle = ptr::null_mut();

        // SAFETY: Passing valid pointers, allocating new C memory that we own
        let result = unsafe {
            crate::bindings::lib_vec3_create(x, y, z, &mut handle_ptr)
        };

        match result {
            LibError::LIB_OK => {
                if handle_ptr.is_null() {
                    // C code returned OK but gave us NULL — treat as error
                    Err(FfiError::NullPointer)
                } else {
                    Ok(Self { raw: handle_ptr })
                }
            }
            LibError::LIB_ERROR_ALLOCATION => Err(FfiError::AllocationFailed),
            LibError::LIB_ERROR_INVALID_INPUT => Err(FfiError::InvalidInput(
                format!("invalid coordinates: ({}, {}, {})", x, y, z)
            )),
            _ => Err(FfiError::Unknown(result as i32)),
        }
    }

    /// Returns the x coordinate.
    ///
    /// # Safety
    ///
    /// `self.raw` must be valid and non-null. Caller must ensure
    /// the Vec3 is not used after being dropped.
    pub fn x(&self) -> f64 {
        // SAFETY: self.raw is guaranteed non-null and valid by construction.
        // The drop implementation ensures memory is not freed while references exist.
        unsafe { (*self.raw).x }
    }

    /// Returns the vector length.
    ///
    /// # Safety
    ///
    /// `self.raw` must be valid and non-null.
    pub fn length(&self) -> f64 {
        // SAFETY: Valid non-null pointer passed to C function that only reads
        unsafe { crate::bindings::lib_vec3_length(self.raw) }
    }

    /// Returns the raw pointer for FFI boundary crossing.
    pub fn as_ptr(&self) -> *const Vec3Handle {
        self.raw
    }
}

/// SAFETY: Vec3Wrapper owns exclusive access to its raw pointer and
/// manages memory via lib_vec3_destroy in Drop. The underlying C library
/// is thread-safe for read-only operations on Vec3.
unsafe impl Send for Vec3 {}
unsafe impl Sync for Vec3 {}

impl Drop for Vec3 {
    fn drop(&mut self) {
        // SAFETY: We have exclusive ownership of this pointer.
        // Drop is called once when Vec3 goes out of scope.
        unsafe { crate::bindings::lib_vec3_destroy(self.raw) }
    }
}
```

## SAFETY Comment Template

Every `unsafe` block must follow this template:

```rust
// SAFETY: [What invariant must hold]
//
// [Why this invariant holds at this call site]
//
// [What undefined behavior occurs if invariant is violated]
unsafe { ... }
```

**Example:**
```rust
// SAFETY: vec_ptr must be non-null and point to a valid Vec3Handle
// allocated by lib_vec3_create. We received LIB_OK from the C function,
// which guarantees allocation succeeded. The Vec3 type owns this pointer
// and will call lib_vec3_destroy in Drop, so the memory will be freed.
//
// If vec_ptr were null, we'd have a null pointer dereference.
let result = unsafe { crate::bindings::lib_vec3_length(self.raw) };
```

## Error Handling at FFI Boundaries

```rust
// src/error.rs
use thiserror::Error;

#[derive(Debug, Error)]
pub enum FfiError {
    #[error("C memory allocation failed")]
    AllocationFailed,

    #[error("invalid input: {0}")]
    InvalidInput(String),

    #[error("null pointer received from C")]
    NullPointer,

    #[error("C function returned unexpected error code: {0}")]
    Unknown(i32),

    #[error("C library error: {0:?}")]
    CLibrary(#[from] crate::bindings::LibError),
}
```

**Rules:**
- Map C error codes to Rust enum variants
- Never let C error codes leak to Rust callers as raw integers
- Include the C error code in the Rust error for debugging
- Validate all return values before use

## CString and CStr

**For string handling at FFI boundaries:**

```rust
use std::ffi::{CString, CStr};

// BAD: assumes ASCII and no nulls
fn pass_string(s: &str) {
    let ptr = s.as_ptr() as *const i8;
    unsafe { c_function(ptr) }
}

// GOOD: convert to CString (null-terminated, owned)
fn pass_string(s: &str) -> Result<(), FfiError> {
    let c_str = CString::new(s)
        .map_err(|_| FfiError::InvalidInput("string contains null byte".into()))?;

    // SAFETY: c_str is valid, null-terminated, owned
    unsafe { c_function(c_str.as_ptr()) };

    Ok(())
}

// Reading strings back from C
fn read_string(ptr: *const i8) -> Result<String, FfiError> {
    if ptr.is_null() {
        return Err(FfiError::NullPointer);
    }
    // SAFETY: ptr is non-null and valid for reading until freed by C
    let c_str = unsafe { CStr::from_ptr(ptr) };
    Ok(c_str.to_string_lossy().into_owned())
}
```

## Panic Boundaries

**Rust panics must not cross FFI boundaries.** A panic in Rust called from C causes undefined behavior.

```rust
// Wrap Rust functions exposed to C in catch_unwind
#[no_mangle]
pub extern "C" fn rust_callback(callback: Option<unsafe extern "C" fn()>) {
    // SAFETY: Callback is a valid C function pointer
    if let Some(cb) = callback {
        let result = std::panic::catch_unwind(std::panic::AssertUnwindSafe(|| {
            // Call the C function — this might call back into Rust
            cb();
        }));

        match result {
            Ok(()) => { /* success */ }
            Err(_) => {
                // Panic caught — C code continues safely
                eprintln!("Rust panic caught in FFI callback");
            }
        }
    }
}
```

## Checklist

```
[ ] build.rs runs bindgen and recompiles C on header changes
[ ] All unsafe blocks have SAFETY comments (invariant / hold / UB)
[ ] Raw pointers are wrapped immediately in safe types
[ ] Null pointers are checked before dereferencing
[ ] CString/CStr used for string conversions
[ ] Memory ownership is explicit (who allocates, who frees)
[ ] Drop impl handles all C memory freeing
[ ] Send + Sync implemented where safe
[ ] Panic boundaries protect C callers
[ ] Error types map C error codes to Rust enums
[ ] No .unwrap() at FFI boundary
[ ] Tests cover: valid input, null pointers, error codes
[ ] cargo audit_unsafe.sh passes
```
